Tag的使用

1.当前分支打Tag

切换到需要打Tag的分支
使用 git tag 命令
如：git tag v1.0
回车后可以使用 git tag 命令查看所有tag
$ git tag

v0.8
v0.9
v1.0
(END)

2.指定commit打Tag

Tag默认打在最后一次commit上，若果需要在某次历史记录上打，需要找到对应的commit id

$ git log –pretty=oneline –abbrev-commit

9b8b62a22 (tag: v0.9) 持卡人订单查询跳转到订单展示页
9659100f5 调整定时任务时间、添加订单号
59f17132c (tag: v0.8) 调用图云2
b8864ea1c 调用图云

这里显示已经打过的两个tag，若果想在commit 9659100f5上添加一个名为“v0.81”的tag
只需使用命令
$ git tag v0.81 9659100f5

再次查看所有tag,
$ git tag

v0.8
v0.81
v0.9
v1.0
(END)

3.创建带有注释的标签

用-a指定标签名，-m指定说明文字：
$ git tag -a v0.7 -m “add tuyun for ctrip” b8864ea1c

4.查看标签信息

标签并不是按时间顺序列出，而是按字母排序的。可以使用 git show 查看标签信息：
$ git show v0.7

tag v0.7
Tagger: admin admin@yeah.net
Date: Thu Nov 29 10:42:25 2018 +0800

add tuyun for ctrip

红色字体是标签注释
